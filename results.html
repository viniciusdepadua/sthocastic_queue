Introduo

Em um pronto-atendimento oftalmolgico h 2 consultrios mdicos. A clnica abre s 9h e os pacientes chegam de acordo com um processo de Poisson com taxa 10min, ou seja, este o tempo esperado de chegada do primeiro cliente aps a abertura da clnica. Os pacientes podem ser atendidos indiscriminadamente por qualquer um dos plantonistas. Ao chegar na clnica, o paciente aguarda at que um dos mdicos esteja disponvel. A durao de cada consulta uma varivel aleatria uniformemente distribuda entre 5 e 20 minutos. A clnica deixa de admitir novos pacientes s 16h e fecha aps terminado o atendimento do ltimo paciente.

PARTE 1: Estratgia de Soluo Computacional

Para simular a fila, faz-se necessrio o controle do estado do sistema entre cada instante que acontece um evento. Um evento aqui definido como um acontecimento de chegada ou sada de cliente. O estado do sistema definido pelo nmero de clientes na fila, nmero de clientes sendo atendidos e nmero de clientes que j foram atendidos. Cada processo de final de atendimento em cada consultorio, chegada de cliente, sada de cliente ser marcado em variveis discretas.

Toda vez que um cliente chega, ser marcado esse tempo de chegada e ento ser verificado se h consultrio livre. Marcam-se os acontecimentos de chegada por ordem de chegada dos pacientes para no final termos o tempo mdio de espera. Alm disso marcaremos o tempo de cada acontecimento, para obter o tempo a mais que o consultorio fica aberto, ento, basta subtrair esse tempo do tempo total que a clinica fica aberta.

PARTE 2: Soluo via Simulao de Monte Carlo

Soluo Computacional

Escreva um programa em R para simular este processo apenas uma vez e responda s seguintes questes:

  1. Quantos pacientes vieram at a clnica?
  2. Quantos pacientes tiveram que aguardar para serem atendidos?
  3. Qual o tempo mdio de espera?
  4. A que horas a clnica fechou?

O cdigo abaixo baseado na estratgia computacional acima descrita.

# Limpa a rea de trabalho
Respostas <- function()
{
  rm(list = ls())
  lambda <- 1/10
  duracao <- 60*7 # de 9 a 16 se passam 7 horas
  t <- 0 #tempo percorrido
  n <- 0 #nmero de pessoas no sistema (fila + atendimento)
  ta <- c(rexp(1,rate = lambda)) #tempo de entrada de cada paciente por ordem de chegada
  td = NULL #tempo de partida de cada paciente por ordem de chegada
  tld <- c(Inf,Inf) # tempo da ultimo partida dos dois consultrios
  finished <- FALSE
  tempoMedioDeEspera = 0
  tempoFechou = 0 #tempo que o pronto atendimento terminou os atendimentos
  numeroPacientes = 0
  aguardou=0
  
  while(!finished){
    #As duas primeiras condicionais servem para ver a ordem dos acontecimentos, executando o cdigo nessa ordem
    if ((ta[numeroPacientes+1] <= tld[1])&&(ta[numeroPacientes+1] <= duracao)) {
      t <- ta[numeroPacientes+1]
      numeroPacientes <- numeroPacientes+1
      n <- n+1
      #Claro que se um conaultrio estiver desocupado quando o paciente chegar ele ir direto ao consultrio
      if (n == 1 || n == 2) {
        tld[n] = t + runif(1, min = 5, max = 20)
        td <- c(td, tld[n])
        #Mantem-se sempre os tempos dos ultimos que sairam atualizados em ordem do menor para o maior
        tld = sort.int(tld)
        
        }
    ta <- c(ta, t + rexp(1,rate = lambda))
  }
  else if ((tld[1] < ta[numeroPacientes+1])&&(tld[1] <= duracao)) {
    t <- tld[1]
    n <- n-1
    #Se aps a sada dessa pessoa tiver apenas uma ou nem uma pessoa, ento o consultrio fica vago e na proxima iterao no h prospeco de sada de paciente dele
    if (n == 0 || n == 1){
      tld[1] <- Inf
      tld = sort.int(tld)
    }
    else {
      td <- c(td, t + runif(1, min = 5, max = 20))
      aguardou=aguardou+1
    }
  }
    #Se um dos tempos ultrapassar o horrio de durao ento no h mais chegadas e repete-se o processo at no haver mais pacientes sobrando
  else if ((min(ta[numeroPacientes+1],tld[1]) > duracao) && (n > 0)) {
    while(n>0){
      t <- tld[1]
      n <- n-1
      if (n > 1) 
        {
          tld[1] <- t + runif(1,min = 5.0, max = 20)
          td = c(td,tld[1])
          tld = sort.int(tld)
      }
      if(n==1)
      {
        tld[1] = Inf
        tld = sort.int(tld)
      }
    }
  }
  else {
      tempoFechou = max(t-duracao,0)
      finished = TRUE
    }
  }
  tempoMedioDeEspera=mean(td-ta[1:numeroPacientes])
  retorno=c(numeroPacientes,aguardou,tempoMedioDeEspera,tempoFechou)
}

Analisando os histogramas, possvel notar que o atendimento tem uma qualidade boa. Pela anlise dos intervalos de confiana da mediana, vemos que os valores de clientes se concentram perto da mediana esperada de 42 clientes, para o tempo total de 7h, considerando a taxa do processo de poisson dado. Apesar do relativamente alto nmero de pacientes que chegaram na clnica, a quantidade desses que aguardaram durante algum tempo na fila tem uma mediana de 10~11 clientes. Essa quantidade , de certa forma, baixa, considerando que apenas em torno de \(\frac{1}{4}\) dos clientes teve que esperar em alguma eventual fila. Alm disso, vemos que o tempo mdio de espera possui uma mediana em torno de 13.625 minutos. Esse tempo , de certa forma, bom, j que um paciente pode demorar at 20 minutos em um consultrio. Ademais, a efetividade da clinica se reflete na mediana dos horarios de fechamento da clinica. Vemos que o tempo que a clinica fica aberta a mais que o esperado de apenas 3~5 minutos. Dessa forma, afirma-se que a qualidade do atendimento da clnica boa.
